JavaScript Class Structure for Design Data

These pages provide reference documentation on the JavaScript classes used to support access to objects and information in designs.

  • DWArray - Built in type
  • DWAttribute - A DWAttribute object represents a single attribute value on DesignWorks entity such as a device or signal.
  • DWCircuit - A DWCircuit object represents a circuit in a DesignWorks design. In a typical, flat circuit (i.e. non-hierarchical), there will just be one circuit object which will contain all other elements (devices, signals, etc.). In hierarchical designs, there is a topmost circuit, sometimes referred to as the "design", even though it is the same type of object. This topmost circuit may contain device instances which themselves contain subcircuits, which can also be represented by this type of object. The DWCircuit object is usually the starting point for getting access to other entities in a design, such as devices and signals.
  • DWDevice - A DWDevice object represents a device instance in a DesignWorks circuit. A variety of methods are available for getting and setting device attributes and other internal values. DWDevice objects are not created directly, but are returned by methods of other objects such as DWCircuit.addDevice.
  • DWLibrary - A DWLibrary object represents a DesignWorks symbol library.
  • DWObject - Built in type
  • DWPage - The DWPage object represents a page in a DesignWorks circuit. The DWPage object is used to get access to visual information about the page, such as border size and style, and can be used to get lists of devices and signals that exists on a specific page.

    A DWPage object is not created directly, but is returned by methods of DWCircuit, such as DWCircuit.pages.

  • DWPic - A DWPic object represents a device symbol or miscellaneous graphic.
  • DWPin - The DWPin object represents a device pin in a DesignWorks circuit. The DWPin object is used to get access to information about the pin, such as pin attributes and the attached device and signal objects.

    A DWPin object is not created directly, but is returned by methods of DWDevice and DWSignal, such as DWDevice.pins.

  • DWSignal - A DWSignal object represents a device instance in a DesignWorks circuit. A variety of methods are available for getting and setting device attributes and other internal values. DWSignal objects are not created directly, but are returned by methods of other objects such as DWPin.signal.
  • DWStatic - This is a static, top-level object that holds some general methods for access to DesignWorks circuit data.
  • DWType - The DWType object represents a device type definition and be obtained from a DWLibrary object or from a device instance (DWDevice) in a circuit.
  • DWTypePin - The DWTypePin object represents a pin on a device type definition. The DWTypePin object is used to get access to information about the pin, such as default pin attributes, pin position, function and direction.

    A DWTypePin object is not created directly, but is returned by methods of DWType, such as DWType.pins

    .

    Modifications are not allowed to type definitions that are currently in use in a design. The methods and properties that cause changes to the pin definition can only be used on type objects that have just been created or read from a library and are not in use yet.

  • SPAddressBook - This object represents a NetPad ideas file and can be used to read, create and modify ideas.
  • SPArray - This is used to return arrays of objects.
  • SPDatabase - This object represents a database created by Microsoft Access or a similar database product and allows access to records within the database. New SPDatabase objects are created using the SPDatabase top-level method.
  • SPFile - An SPFile object is used to work with files and directories, either on your local system or on remote Internet sites. Here are some notes regarding SPFile objects:
  • An SPFile object can be created directly using the SPFile top-level function or can be generated by other methods such as getFileList
  • In this version, the file manipulaton methods, such as read, write, delete, etc. are implemented only for local files (i.e. on your own computer). SPFile objects can be used to store information on files on remote FTP sites (see FTP.getFileList), but no active operations can be done on them.
  • It is not normally necessary to explicitly open and close files. If you perform an operation like read or write that requires a file to be open, it will be opened automatically. Files are closed again automatically when the object is destroyed. See the open and close methods for notes on cases where you may want to control this explicitly.
  • SPFTP - An SPFTP object is used to connect to an FTP site and perform uploads, downloads, and other FTP operations. An SPFTP object is created using the SPNet.SPFTP method.
  • SPHTMLDialog -

    This class implements a dialog box that allows information to be displayed or the user to be prompted for information using HTML formatted pages. If only a single HTML page is specified, a simple dialog box is displayed. If multiple pages are specified, then a tabbed "property sheet" box is displayed. Methods are provided to allow controls embedded in the pages to be initialized and their final values read back after the dialog is closed.

    Instances of this class are created using the SPHTMLDialog top-level function.

  • SPInternet - This object is used to access files from the Internet. NOTE: This version supports only the HTTP protocol, i.e. any URL starting with "http". An SPInternet object is created using the Net.SPInternet method.
  • SPNet - This is a static, top-level object used to create connections to HTTP (World Wide Web) and FTP servers and to perform various related tasks. The SPNet object always exists and is never explicitly created or destroyed. Since it is a top-level object, methods in this object do not have to be preceded with "Net.".
  • SPRect - Built in type
  • SPScript - This is a global, top-level object containing a collection of general purpose methods and properties. This means that the SPScript object is never created or destroyed and also that the name Script does not need to be used as a prefix to these items. For example the "halt" method can be used just as "halt()". You can refer to it as "SPScript.halt()" if desired, but it is not necessary.
  • SPStringMap - An SPStringMap is a table that allows you to store and look up values based on a string index. It can be used, for example, to keep values associated with names or keep lists of strings to determine when an item already exists in the list.
  • SPZip - This object lets you create, add to, and extract from standard Zip archive files. An object of this class is created by calling the top-level function SPZip.
